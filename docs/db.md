# Database Schema

LinComms uses **PostgreSQL with Prisma ORM**. The schema enforces invite-only onboarding and role-based access.

## Entities

### User
- `id` — unique identifier (primary key)
- `email` — unique user email
- `role` — `ADMIN` or `MEMBER`
- `hashedPassword` — secure password hash
- `invitesCreated` — invites generated by this user
- `acceptedInvite` — invite used to join LinComms

### Invite
- `id` — unique identifier
- `token` — unique invite token
- `email` — optional restriction to a specific email
- `role` — role assigned upon acceptance
- `createdBy` — the user (Admin) who generated the invite
- `acceptedBy` — the user who redeemed the invite
- `expiresAt` — expiration timestamp
- `used` — boolean flag for invite validity

## Relations
- **One-to-many:** A user can create multiple invites.
- **One-to-one:** An invite can only be used by a single user.

---

## Security Considerations
- All passwords are hashed with bcrypt.
- Invites are cryptographically signed to prevent tampering.
- Expired/used invites are invalidated automatically.
